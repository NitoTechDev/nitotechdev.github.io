What is fuzzing?

Jan 11, 2026 | Concept study on Fuzzing

Introduction

Recently I started taking a course on vulnerability research. Specifically on learning how to find and potentially exploit Zero-days. This course has been super fun so far, and it's nice to have structured resources available to learn from. One of the concepts I learned about recently was grammar-based fuzzing. I had heard of fuzzing in the past but it wasn't anything that I was familiar with enough to understand how it's used. So in an attempt to gain a better understanding, I'm writing this blog post!

Below, you will find my research on fuzzing, primarily on what is fuzzing, what is it used for, and what kind of software does it target.

Please hover over any words highlighted in purple to get a description of what they mean!

What is fuzzing?

Fuzzing, is a software testing technique that assist in finding vulnerabilities and defects in a given software. This is done through providing random inputs into the program to try and achieve a specific outcome. These outcomes can range from a software crashing, providing output, privilege escalation, and more.

With that in mind lets talk about one of the most important pieces of fuzzing, input.

What is input in fuzzing?

Input, is what the fuzzing tool is providing to the software in order to produce an outcome. Input can come in many forms, it can be completely random characters, malformed input, syntax specific output and more.

Input is determined by the type of fuzzing you want to perform. A few examples are grammar-based, coverage-guided, and unguided fuzzing. Each type of fuzzing is used for specific kinds of environments and uses different kinds of input.

  • Grammar-based fuzzing - Improves test/code coverage by specifying the grammar of semi-valid inputs. Uses syntax specific inputs.

  • Coverage-guided fuzzing - Improves each itteration of testing by making informed decision on each tests output. Guiding the input to move closer to the desired outcome for that program.

  • Unguided fuzzing - Unlike other forms of fuzzing, this fuzzing technique takes completly random input as well as malformed input. Not relying on any guidance or specific coverage techniques.

What is fuzzing used for?

Fuzzing is used for several things. It can help software developers with finding potential security risks they haven't thought of yet. It assists with validating input fields to prevent exploitation. When it comes to cybersecurity, fuzzing is used by red teamers, blue teamers, and malcious hackers alike to attempt to find security vulnerabilities and zero days in a given application.

Since fuzzing is primarily an automative technique, it saves time for developers and cybersecurity practitioners alike while unconvering bugs that other testing methods may not be able to find.

What kinds of software does fuzzing target?

Fuzzing tools test softwares that rely on user and external inputs. This can include a variety of software, such as:

  • Web / Mobile apps
  • Operating systems
  • APIs
  • PDF readers
  • Compilers
  • and more...

Depending on the type of software you are targeting you'd use a respective fuzzing technique. As mentioned earlier, if you were attempting to fuze a C# compiler. You'd then want to rely on a grammar-based fuzzer that uses syntax specific input so you don't waste time with input that wouldn't work with the compiler in the first place.

Conclusion
